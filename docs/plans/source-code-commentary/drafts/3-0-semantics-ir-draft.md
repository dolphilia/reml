# 第3部: 意味論と中間表現

**目的**: 検証と変換について説明すること。

型チェックを通過したプログラムは、もはや単なる構文木ではありません。それは実行可能な「意味」を持った有機的な存在です。第3部では、フロントエンドの仕上げとして行われる意味解析と、実行時パイプラインへの橋渡しに焦点を当てます。

名前解決によって識別子を結びつけ、エフェクトシステムによって副作用を管理し、ストリーミングパイプラインによってデータを流す。ここでは、静的な構造が動的な振る舞いへと変換される過程で、Remlがどのような中間表現を用い、どのような検証を行っているのかを見ていきます。言語の設計思想が色濃く反映される、コンパイラの「頭脳」とも言える領域です。

## 本部の構成

- **第8章: 意味解析 (Semantic Analysis)**
  - 名前解決、スコープ管理、そして型付きASTの整合性チェックなど、プログラムの意味的な正当性を検証します。
- **第9章: 実行パイプライン**
  - Remlの特徴であるストリーミング実行を支える、パイプライン処理のデータフローと制約を解説します。
- **第10章: エフェクトとFFI実行**
  - 純粋な計算と外界への作用（副作用）を分かつエフェクトシステムと、外部関数呼び出し（FFI）の安全な取り扱いについて説明します。
