# 第6部: ツールと周辺領域

**目的**: 周辺ツールと補助的なコードを整理すること。

コンパイラ本体やランタイムの周囲には、それらを支え、活用するための重要なエコシステムが存在します。第6部では、プラットフォームの差異を吸収するアダプタ層、他言語との連携を自動化するツール、そして開発プロセス自体を支援するユーティリティ群にスポットライトを当てます。

これらは目立たない存在かもしれませんが、Remlの実用性とポータビリティを支える縁の下の力持ちです。クロスプラットフォーム対応や、複雑なFFIバインディング生成の裏側にある工夫を知ることで、Remlというシステム全体の堅牢さを理解できるでしょう。

## 本部の構成

- **第19章: Adapter レイヤ**
  - OSや環境ごとの差異（ファイルシステム、ネットワーク、時間など）を吸収する抽象化レイヤの設計を解説します。
- **第20章: FFI Bindgen**
  - 外部ライブラリの定義からRemlのバインディングコードを自動生成するツールの仕組みを紹介します。
- **第21章: 開発支援ツール**
  - ビルドタスク管理（`xtask`）や、コードベースの監査・維持に用いられる内部ツールのワークフローを説明します。
