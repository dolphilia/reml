# 第2部: フロントエンド（解析）

**目的**: テキストがどのように意味になるかを説明すること。

プログラミング言語にとって、最も人間的な側面と最も機械的な側面が交錯する場所、それがフロントエンドです。第2部では、開発者が記述したテキストデータが、コンパイラによってどのように解釈され、構造化され、検証されるのかを詳細に追跡します。

一文字一文字の読み込みから始まり、文法構造の構築、そして厳密な型システムによる整合性の検証に至るまで。このプロセスは、混沌とした文字列に秩序を与え、プログラムとしての「意味」を確定させるための戦いでもあります。Remlがどのようにしてコードの意図を汲み取り、あるいは誤りを指摘するのか。その精緻なメカニズムを解き明かしていきましょう。

## 本部の構成

- **第4章: 字句解析 (Lexical Analysis)**
  - ソースコードをトークンの列へと分解する、最初の変換プロセスを扱います。
- **第5章: 構文解析 (Parsing)**
  - トークン列から抽象構文木（AST）を構築し、プログラムの構造を決定する方法を解説します。
- **第6章: 診断と出力**
  - エラーや警告をユーザーに親切に伝えるための、診断レポートの仕組みについて詳述します。
- **第7章: 型チェックと型推論**
  - プログラムの安全性を保証する要塞、型システムの内部動作と推論アルゴリズムに迫ります。
