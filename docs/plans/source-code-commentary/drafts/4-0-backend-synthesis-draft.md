# 第4部: バックエンド（合成）

**目的**: 実行可能コードを生成する方法を説明すること。

中間表現として抽出されたプログラムの意味は、ここで最終的な「形」を与えられます。第4部では、抽象的な命令列を、実際のハードウェア上で動作する機械語へと変換するバックエンドの役割に焦点を当てます。

LLVMという強力な基盤の上に、Reml独自の型システムやメモリモデルをどのようにマッピングするのか。そして、生成されたコードが必要とするランタイム機能とどのようにリンクし、整合性を検証するのか。理論上の計算が、現実のプロセッサで駆動する電気信号へと変わる、そのダイナミックな「合成」のプロセスを探求します。

## 本部の構成

- **第11章: LLVMへのラウアリング**
  - Remlの中間表現からLLVM IRへの変換、メモリレイアウト、ABIの決定プロセスを解説します。
- **第12章: ランタイム連携と検証**
  - 生成コードとランタイムライブラリのリンク、およびターゲットアーキテクチャに依存した検証手順について説明します。
