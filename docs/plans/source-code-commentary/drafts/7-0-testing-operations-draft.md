# 第7部: テストと運用

**目的**: 品質保証と運用手順を整理すること。

コードを書くことは終わりのないプロセスの始まりに過ぎません。第7部では、Remlコンパイラの品質を長期にわたって維持し、進化させ続けるための「守り」と「運用」の戦略を共有します。

ユニットテストから統合テストに至る階層的なテスト戦略、そして仕様書（Spec）と実装の乖離を防ぐための同期プロセス。これらは地味ながらも、コンパイラという複雑なシステムの信頼性を担保するために不可欠な活動です。Remlがどのようにして自らの正しさを証明し続けているのか、その舞台裏を明かします。

## 本部の構成

- **第22章: テスト戦略**
  - 単体テスト、統合テスト、そしてエコシステムテストまで、多層的なテストの目的と実装方法を解説します。
- **第23章: 仕様との同期**
  - 言語仕様書（`docs/spec`）とコンパイラ実装の一貫性を保つためのプロセスと、その更新手順について説明します。
