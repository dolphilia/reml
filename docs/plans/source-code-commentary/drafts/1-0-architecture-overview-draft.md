# 第1部: アーキテクチャ概要

**目的**: 道を歩む前に、読者に領域の地図を与えること。

壮大な建築物を理解するには、まずその全体像を眺め、主要な区画とそれをつなぐ通路を知ることが不可欠です。第1部では、Remlコンパイラという巨大なシステムの「鳥瞰図」を提供します。

ソースコードというただの文字列が、いかにして計算機が理解可能な命令へと変貌を遂げるのか。その長い旅路の出発点として、まずはコンパイラ全体を貫くパイプラインの流れを追い、このリポジトリを構成する主要なモジュール群の役割を整理します。

ここでの理解は、以降の各論へ深く潜っていくための羅針盤となるはずです。

## 本部の構成

- **第1章: Remlコンパイルパイプライン**
  - ソースコードからバイナリまでのデータの流れをハイレベルな視点で俯瞰します。
- **第2章: リポジトリ構造**
  - `compiler/` ディレクトリ配下に広がる `frontend`, `backend`, `runtime` などの主要ディレクトリの地図を描きます。
- **第3章: 実行の入口**
  - エントリポイントである `reml_frontend` と `remlc` が、どのようにしてコンパイラを起動し、設定を受け付け、診断結果を出力するのかを解説します。
