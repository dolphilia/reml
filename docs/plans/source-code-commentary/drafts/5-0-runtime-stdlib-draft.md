# 第5部: ランタイムと標準ライブラリ

**目的**: 実行環境について説明すること。

コンパイラが生成したコードは、真空の中に存在するわけではありません。それは「ランタイム」という豊饒な土壌の上で初めて動作します。第5部では、Remlプログラムの実行を支えるランタイムシステムと、開発者に提供される標準ライブラリの内部実装を解剖します。

メモリ管理、入出力、並行処理といった基本的な機能から、Capabilityによる権限管理、DSLサポート、そしてFFIによる外部連携まで。Remlのランタイムは単なるライブラリの集合体ではなく、言語の哲学（安全性、監査可能性、拡張性）を体現するプラットフォームそのものです。その深層構造を明らかにしましょう。

## 本部の構成

- **第13章: ランタイムの全体像**
  - ランタイムの起動、初期化、プラグインシステムなどのアーキテクチャ概要を示します。
- **第14章: Capability と監査**
  - セキュアな実行環境を実現するための権限管理（Capability）と、操作ログの監査機構について詳述します。
- **第15章: 標準ライブラリのプリミティブ**
  - コレクション、IO、テキスト処理など、標準ライブラリの中核をなすデータ構造とAPIの実装を解説します。
- **第16章: 解析・DSL・診断**
  - 言語内DSLの構築や、ランタイム時に発生する診断情報の処理機構について扱います。
- **第17章: FFI とネイティブ連携**
  - 他言語やネイティブコードとの境界、メモリ共有、ABIの整合性確保について説明します。
- **第18章: LSP/システム補助**
  - 言語サーバープロトコル（LSP）のサポートや、OSシステムコールへの抽象化レイヤについて触れます。
