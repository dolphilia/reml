# 0.3 ロールバック計画

## 想定リスク
- 更新した Rust で `compiler/rust/` の一部クレートがビルド不能になる。
- 依存クレートの更新が想定外の破壊的変更を含む。

## ロールバック方針
- `rustup` で前の安定版に戻し、`Cargo.lock` を更新前の状態へ戻す。
- 依存クレートが原因の場合は、`cargo update -p <crate>@<version>` でピン留めする。
- ロールバック実施の判断と手順を `reports/spec-audit/summary.md` に記録する。

## 例外対応
- どうしても最新ツールチェーンが必要な場合は、特定のクレートにパッチを当てるか、最小限の修正で互換を確保する。
