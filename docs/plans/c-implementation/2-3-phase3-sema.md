# フェーズ 3: 意味解析 (Sema)

このフェーズでは、意味的検証、名前解決、型チェックを実装します。

## 3.1 シンボルテーブルとスコープ管理
- **データ構造**: `UTHASH` ベースのシンボルハッシュマップ。
- **スコープ**: デックのスタック、または永続データ構造アプローチ。
- **タスク**:
  1.  `Symbol` 構造体 (種類: Func, Var, Type) の定義。
  2.  `enter_scope()` と `exit_scope()` の実装。
  3.  シャドーイングをサポートする `lookup_symbol()` の実装。

## 3.2 名前解決
- **目標**: AST 識別子をその定義にリンクする。
- **タスク**:
  1.  AST ウォーク (C での Visitor パターン実装)。
  2.  変数と型の解決。
  3.  「未定義の識別子」エラーの報告。
  4.  モジュール/インポートの処理 (基本的なファイル検索)。

## 3.3 型システムの表現
- **構造体**: `include/reml/typeck/` 内の `Type` enum/union。
  - プリミティブ: `Int`, `Float`, `Bool`, `Char`, `Unit`.
  - 複合: `Struct`, `Enum`, `Tuple`, `Function`, `Array`.
- **メモリ**: ポインタ比較による等価性チェックのための型インターニング (Arena または Hash Consing)。
- **ライブラリ**: インターニングテーブル用の `uthash`。

## 3.4 型チェック (推論と検査)
- **仕様**: `docs/spec/1-2-types-Inference.md`。
- **戦略**: 双方向型検査 (Bidirectional Type Checking) (または HM スタイルの単一化)。
- **タスク**:
  1.  `unify(t1, t2)` の実装。
  2.  `infer(expr) -> Type` の実装。
  3.  `check(expr, expected_type)` の実装。
  4.  リテラルの多相性処理 (例: `1` は `Int` または `Float` または `BigInt`)。

## 3.5 効果分析 (予備的)
- **仕様**: `docs/spec/1-3-effects-safety.md`。
- **目標**: 関数境界での基本的な効果 (IO, State) の追跡。
- **タスク**:
  1.  効果注釈の解析。
  2.  コールグラフ上の効果伝播 (ナイーブな実装)。

## 3.6 検証
- **ユニット機能**:
  - 識別子解決: 使用箇所が宣言を指していることを確認。
  - 型チェック: `1 + 1` は有効、`1 + "a"` は無効。
- **テスト**:
  - `tests/unit/test_sema.c`: モック AST を使用して型結果をアサート。
  - 統合: 有効/無効な Reml スニペットを実行し、終了コード/エラーメッセージを確認。
