# Reml C言語実装計画: 概要

本文書は、C言語によるRemlプログラミング言語の新規実装計画の概要です。

## 動機

現在RemlにはOCamlとRustの実装が存在しますが、以下の理由からC言語版の実装が求められています：

- **移植性**: C言語は依然として最も移植性が高く、新しいプラットフォームでのブートストラップが容易です。
- **統合**: LLVMやその他の低レベルライブラリとの親和性が高いです。
- **透明性**: RustやOCamlと比較してよりプリミティブな言語であるため、「ブラックボックス」な抽象化が少なく、特定の低レベルデバッグや理解に有益です。
- **チームの専門性**: 開発チームの強力なC言語のバックグラウンドを活用できます。

## コア目標

1.  **仕様準拠**: Remlの仕様書 (`docs/spec/`) を完全にサポートすること。
2.  **実行能力**: `examples/practical/` および `examples/spec_core/` にあるサンプルを実行可能にすること。
3.  **クロスプラットフォーム**: macOS, Linux, Windows をサポートすること（初期開発は macOS で行います）。
4.  **保守性**: CMake を使用したクリーンでモジュール化されたコード構造を維持すること。

## 技術スタックと制約

-   **言語**:
    -   **C**: C11 (ISO/IEC 9899:2011) または C17。これを主要言語とします。
    -   **C++**: C++17。特定のライブラリや機能のために必要な場合のみ使用し、メインコードベースとの互換性を保つために `extern "C"` でラップします。
-   **ビルドシステム**: CMake (推奨バージョン 3.10以上)。
-   **コンパイラ**:
    -   **macOS**: Clang (デフォルト)
    -   **Linux**: GCC
    -   **Windows**: MSVC
-   **スクリプト**: 自動化タスクには Python (`.py`) または Shell (`.sh`) を使用。
-   **テスト**: CTest (CMakeに統合)。

## 実装哲学

-   **ライブラリ選定**: コモンタスク（コンテナ、JSONなど）には、車輪の再発明を避けるために、信頼性が高く十分にテストされたサードパーティ製ライブラリを使用します（適切なものが存在しない場合を除く）。
-   **C言語ファースト**: コードベースは主にC言語であるべきです。C++は（LLVMのような）強力なライブラリのための「脱出口」であり、デフォルトではありません。
-   **透明性**: 過剰なエンジニアリングを避けます。抽象化を明確にし、問題領域に直接マッピングさせます。
