# 4.0 エコシステム仕様 概要

## 概要
エコシステム章は Reml の配布・運用・コミュニティ体制を統合的に扱い、パッケージ管理からガバナンスまでのプロセスを整理します。CLI やレジストリ、ツールチェーンを通じて開発者が一貫した体験を得られるようにしつつ、コミュニティ活動・指標管理・リスク対応を横断して設計判断をまとめています。

## セクションガイド
- [4.1 パッケージマネージャと CLI](4-1-package-manager-cli.md): `reml` CLI のサブコマンド体系、マニフェスト統合、クロスコンパイル支援と監査要件を定義します。
- [4.2 レジストリと配布](4-2-registry-distribution.md): レジストリの API 境界、パッケージメタデータ、公開ワークフローとセキュリティポリシーを整理します。
- [4.3 開発ツールチェーン](4-3-developer-toolchain.md): LSP・フォーマッタ・テストランナーなどのツール群、配布/ターゲット管理、AI 支援ツールの責務をまとめます。
- [4.4 コミュニティとコンテンツ戦略](4-4-community-content.md): 公式サイト構成、コミュニティ運営原則、コンテンツ戦略と KPI 設計を扱います。
- [4.5 ロードマップと指標管理](4-5-roadmap-metrics.md): フェーズ別ロードマップ、技術/コミュニティ指標、リスクトラッキングとレポーティング手順を定義します。
- [4.6 リスクとガバナンス](4-6-risk-governance.md): リスク分類と軽減策、ガバナンスモデル、監査・コンプライアンス手順、継続的改善サイクルをまとめます。
