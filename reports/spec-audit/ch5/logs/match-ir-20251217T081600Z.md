# CH1-MATCH LLVM IR ダンプ（Constructor payload / 式評価の拡張）(20251217T081600Z)

本ログは `tmp/` 配下のフロントエンド MIR JSON を入力に、Rust LLVM バックエンドの `llvm_ir`（LLVM 風）をダンプした結果です。

今回の目的は次の 2 点を「再現可能な形」で固定することです。

1. `Some(x)` / `Ok(x)` / `Err(x)` のような **Constructor の内側パターン**で、payload を取り出して下位パターンへ伝搬できること
2. ガード/本体で `FieldAccess` / `Call` / `Binary`（例: `%` / `+`）を最小限評価し、`match_result <- #...` のフォールバックを減らせること

- `MIR_PATH="$(pwd)/tmp/mir-bnf-matchexpr-result-guard-else-ok.json" cargo test --manifest-path compiler/rust/backend/llvm/Cargo.toml dump_llvm_ir_from_mir_path -- --ignored --nocapture`

## tmp/mir-bnf-matchexpr-option-canonical.json

`Some(n)` の payload 抽出が `arm0.ctor_payload` ブロックとして現れ、`n.to_string()` と `"value=" + ...` が `@reml_field_access` / `@reml_call` / `@reml_str_concat` へ落ちていることを確認できます。

```
fn describe:
define {i8*, i64} describe(ptr) {
arm0.pat:
  ; ctor_check(Some, args=1 on {"name":"value","span":{"end":179,"start":174}})
  ; ctor_check(Some) on {"name":"value","span":{"end":179,"start":174}} -> non-null then arm0.ctor_payload else arm1
  %ctor1 = icmp ne ptr %value, null
  br i1 %ctor1, label %arm0.ctor_payload, label %arm1
arm0.ctor_payload:
  ; payload({"name":"value","span":{"end":179,"start":174}}.Some) <- {"name":"value","span":{"end":179,"start":174}}
  %payload2 = call ptr @reml_ctor_payload_Some(ptr %value)
  ; bind(n)
  br i1 true, label %arm0.body#5, label %arm1
arm0.body#5:
  ; exec body#5
  ; field_access %n.to_string
  %field3 = call ptr @reml_field_access(ptr %n, ptr "to_string")
  %call4 = call Str @reml_call(ptr %field3)
  %concat5 = call Str @reml_str_concat(Str "value=", Str %call4)
  ; match_result <- expr#5 (%concat5)
  %match_result6 = call Option<Int> @reml_value(Option<Int> %concat5)

}
```

## tmp/mir-bnf-matchexpr-result-guard-else-ok.json

`Ok(n) when n % 2 == 0` のガードが `srem i64` + `icmp eq` として出力され、`Err(message)` 側の `"error:" + message` が `@reml_str_concat` に落ちていることを確認できます。

```
fn classify:
define {i8*, i64} classify(ptr) {
arm0.pat:
  ; ctor_check(Ok, args=1 on {"name":"input","span":{"end":266,"start":261}})
  ; ctor_check(Ok) on {"name":"input","span":{"end":266,"start":261}} -> then arm0.ctor_payload else arm1
  %ctor1 = call i1 @reml_is_ctor_Ok(ptr %input)
  br i1 %ctor1, label %arm0.ctor_payload, label %arm1
arm0.ctor_payload:
  ; payload({"name":"input","span":{"end":266,"start":261}}.Ok) <- {"name":"input","span":{"end":266,"start":261}}
  %payload2 = call ptr @reml_ctor_payload_Ok(ptr %input)
  ; bind(n)
  br i1 true, label %arm0.guard#5, label %arm1
arm0.guard#5:
  ; guard arm0.guard#5 -> arm0.body#6/arm1
  %mod3 = srem i64 %n, 2
  ; guard compare op=== lhs=%mod3 rhs=0
  %guard4 = icmp eq i64 %mod3, 0
  br i1 %guard4, label %arm0.body#6, label %arm1
...
arm2.ctor_payload:
  ; payload({"name":"input","span":{"end":266,"start":261}}.Err) <- {"name":"input","span":{"end":266,"start":261}}
  %payload10 = call ptr @reml_ctor_payload_Err(ptr %input)
  ; bind(message)
  br i1 true, label %arm2.body#10, label %match11.end
arm2.body#10:
  ; exec body#10
  %concat11 = call Str @reml_str_concat(Str "error:", Str %message)
  ; match_result <- expr#10 (%concat11)
  %match_result12 = call Result<Int, Str> @reml_value(Result<Int, Str> %concat11)
  br label %match11.end
...
}
```
